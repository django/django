==========================
Serializing Django objects
==========================

Django's serialization framework provides a mechanism for translating Django
models into other formats. These formats are typically text-based and are used
to transfer data, but serializers can support any format.

.. seealso::

    To export data from your database, you can use the
    :djadmin:`dumpdata` management command.

Serializing data
================

Basic usage::

    from django.core import serializers

    data = serializers.serialize("json", SomeModel.objects.all())

The ``serialize()`` function takes:

- The output format (e.g., ``json``)
- A :class:`~django.db.models.query.QuerySet`

You can also use a serializer instance::

    JSONSerializer = serializers.get_serializer("json")
    serializer = JSONSerializer()
    serializer.serialize(SomeModel.objects.all())
    data = serializer.getvalue()

To write directly to a file::

    with open("data.json", "w") as output:
        serializer.serialize(SomeModel.objects.all(), stream=output)

.. note::

    Calling :func:`~django.core.serializers.get_serializer` with an unknown
    format raises ``SerializerDoesNotExist``.

Subset of fields
----------------

You can limit serialized fields::

    data = serializers.serialize(
        "json",
        SomeModel.objects.all(),
        fields=["name", "size"],
    )

Only specified fields are included. The primary key is always included as ``pk``.

.. note::

    Deserialization may fail if required fields are missing.

Deserializing data
==================

Example::

    for obj in serializers.deserialize("json", data):
        obj.save()

The ``deserialize()`` function returns ``DeserializedObject`` instances,
which wrap unsaved model instances.

.. note::

    If no primary key is provided, a new object is created.

To ignore unknown fields::

    serializers.deserialize("json", data, ignorenonexistent=True)

Serialization formats
=====================

Supported formats:

==========  ==========================================
Identifier  Description
==========  ==========================================
``xml``     XML format
``json``    JSON format
``jsonl``   JSON Lines format
``yaml``    YAML format (requires PyYAML)
==========  ==========================================

JSON
----

Example output::

    [
        {
            "pk": 1,
            "model": "app.model",
            "fields": {
                "name": "example"
            }
        }
    ]

Foreign keys are represented by primary key values.
Many-to-many relations are represented as lists of primary keys.

Custom JSON encoder::

    from django.core.serializers.json import DjangoJSONEncoder


    class CustomEncoder(DjangoJSONEncoder):
        def default(self, obj):
            return super().default(obj)

Usage::

    serializers.serialize("json", SomeModel.objects.all(), cls=CustomEncoder)

JSONL
-----

Each object is written on a new line::

    {"pk": 1, "model": "app.model", "fields": {...}}
    {"pk": 2, "model": "app.model", "fields": {...}}

This format is useful for large datasets.

YAML
----

Example::

    - model: app.model
      pk: 1
      fields:
        name: example

Custom serialization formats
----------------------------

You can define custom serializers.

Example::

    SERIALIZATION_MODULES = {
        "csv": "myapp.serializers.csv_serializer",
    }

.. note::

    Serializers with ``internal_use_only = True`` are intended for internal
    use and are not available via management commands such as
    :djadmin:`dumpdata`.

    To make a serializer available for external use, set
    ``internal_use_only = False``.
